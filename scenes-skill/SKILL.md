# Scenes

Agent **skill** for interactive educational scenes. Scene content is pretty much pure Three.js in a small portable package. This skill is the form-factor; the npm package also ships the CLI and viewer needed to manage and view scenes.

## Status

Placeholder for the first repo commit. Full static-loop contract, package layout, and validate/show workflow will be drafted with implementation (plan steps 1a–1d). Do not treat this file as complete agent instructions yet.

## Intended loop (preview)

1. User discusses a concept (or model showcase) with the agent.
2. Agent writes/edits a scene package under the local workspace.
3. Agent runs CLI validate (and show when viewing is needed).
4. User explores in the browser viewer (parameters, camera, tools).

Details land when development starts.
